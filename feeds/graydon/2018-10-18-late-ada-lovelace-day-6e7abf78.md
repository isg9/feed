# Late Ada Lovelace day

I missed it this year (sigh, timing so bad) but in recent years I've been reading concurrency, database, type theory and computational logic research, so here are some new rad women in CS I've been following:

2. [Christine Paulin-Mohring](https://www.lri.fr/~paulin/): one of the main forces developing Coq, winner of ACM software systems award. Possibly most influential / mainstream interactive theorem prover in use.

3. [Amal Ahmed](http://www.ccs.neu.edu/home/amal/): central to the past couple decades progress in type theory, pioneer in logical relations and step indexing; everything from typed assembly language / type-preserving compilers to semantics of mutable / recursive types, linear types, verified compilers and inter-language linking, gradual typing.

4. [Véronique Benzaken](https://www.lri.fr/~benzaken/): decades of really serious work formalizing the types and semantics of query languages and databases. If you want to know what SQL, Datalog, XQuery or their underlying data models *mean*, go read [all her papers](https://www.lri.fr/~benzaken/publications.en.html).

5. [Stephanie Weirich](https://www.cis.upenn.edu/~sweirich/): leading the charge for dependent types in mainstream (ish) languages. If you find yourself hacking in dependent Haskell a few years from now, her team is why.

6. [Niki Vazou](https://nikivazou.github.io/): if you find dependent Haskell a bit too cumbersome but you still want better verification tech, you might find her Liquid Haskell system fits the bill. Liquid types express more restricted sets of properties, but the proofs are automatically discharged: much easier!

7. [Heather Miller](http://heather.miller.am/): building and formalizing a slew of language-level distributed and concurrent programming abstractions (monotonic, serializable, convergent, dataflow models), documenting the highs and lows along the way in textbook form.

8. [Nada Amin](https://www.cl.cam.ac.uk/~na482/cv/): [Broke](https://infoscience.epfl.ch/record/227175?ln=en) and then [rebuilt from the ground up](https://infoscience.epfl.ch/record/215280/files/paper_1.pdf) Scala's type system; if you find yourself enjoying the new "dotty" Scala compiler someday, you can thank her for giving it a sound type system to build on.
