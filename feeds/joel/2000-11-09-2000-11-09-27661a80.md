# 2000/11/09

Is Bush going to actually be the next president because of a *usability bug*?

[![picture-ballot: ](https://i0.wp.com/www.joelonsoftware.com/wp-content/uploads/2000/11/Bush_Steals_Florida.jpg?resize=280%2C170&ssl=1)](http://www.salon.com/politics/feature/2000/11/09/florida_vote/index.html)

“If your program model is nontrivial, it’s probably not the user model.” (from [Chapter 2](https://www.joelonsoftware.com/uibook/chapters/fog0000000058.html) of [UI design for programmers](https://www.joelonsoftware.com/uibook/chapters/fog0000000057.html)) Any time you do something even a *little* bit complicated, some part of your user population is going to be confused, because they thought that you were doing the *simple* thing.

Obviously, usability matters for more than just web sites. Tog writes about this in [When Interfaces Kill](http://www.asktog.com/columns/027InterfacesThatKill.html). And this Florida ballot is going to show up in UI books for the next 100 years.

**Florida’s Web Site**

The Florida Department of State web server is bogged down from zillions of people checking election results. State CIO Gene Bryan is [quoted by CNN](http://www.cnn.com/2000/TECH/computing/11/09/fla.election.overload.idg/index.html) as saying, “There’s not a whole lot you can do other than just get through the night and try to regroup in the morning to get some additional resources to handle the workload.”

Um, hello? [They’ve got](http://election.dos.state.fl.us/index.html) 4 or five stupid GIFs and a gratuitous scrolling Java applet on that damn page. No wonder you can’t handle the load. Clue: Turn off the applet and GIFs for a couple of weeks. Duh. Any half-baked web developer would know that.
