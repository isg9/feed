# 2003/04/28

Eric Sink [analyzes](http://biztech.ericsink.com/Abstraction_Pile.html) all the leaky abstractions he just used to create [Vault](http://www.sourcegear.com/vault/index.asp), SourceGear’s new source control system, written in .NET. “I’ve used Java extensively, and I loved the productivity gains we got during the beginning and middle of the development cycle.  But things got ugly at the end. [![](https://i0.wp.com/www.joelonsoftware.com/wp-content/uploads/2003/04/fuji.png?resize=120%2C180&ssl=1)](http://www.amazon.com/exec/obidos/tg/detail/-/0316919160/joelonsoftware) All those layers of abstraction started contributing to our bug list.  I’ve been involved in a couple of projects which completely failed because Java was chosen…. This success stands as a testimony to how incredible .NET really is.”

William Poundstone, who wrote an [excellent book on game theory](http://www.amazon.com/exec/obidos/tg/detail/-/038541580X/joelonsoftware), has a new book about Microsoft interview questions, “ [How Would You Move Mount Fuji? Microsoft’s Cult of the Puzzle](http://www.amazon.com/exec/obidos/tg/detail/-/0316919160/joelonsoftware)”
